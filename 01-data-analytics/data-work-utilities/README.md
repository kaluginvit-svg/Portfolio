# Анализ данных

Веб-приложение для анализа табличных данных из Excel, CSV и PDF файлов с поддержкой визуализации и AI-анализа.

## Возможности

- 📊 Загрузка и парсинг файлов: Excel (.xlsx, .xls), CSV, PDF
- 📈 Базовая аналитика: суммы, уникальные значения
- 📉 Интерактивная визуализация данных (гистограммы, линейные графики, круговые диаграммы, точечные графики)
- 🤖 AI-анализ данных с использованием GigaChat
- 🎨 Современный и чистый интерфейс
- 📱 Полноэкранный режим
- 🔄 Динамическое создание графиков с выбором параметров и типов

## Требования

**Для фронтенда:**
- Node.js 16+ и npm

**Для бэкенда:**
- Убедитесь, что у вас установлен Python 3.8 или выше
- Рекомендуется использовать виртуальное окружение

## Установка и запуск

### Шаг 1: Клонирование репозитория (если нужно)

```bash
git clone <repository-url>
cd <project-directory>
```

### Шаг 2: Установите зависимости бэкенда

```bash
# Создайте виртуальное окружение с Python 3.11 (рекомендуется)
# Важно: используйте Python 3.11 или 3.12, не Python 3.14!
py -3.11 -m venv venv

# Активируйте виртуальное окружение
# Windows PowerShell:
.\venv\Scripts\Activate.ps1
# Если PowerShell блокирует скрипты, сначала выполните:
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
# Windows CMD:
# venv\Scripts\activate.bat
# Linux/Mac:
# source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt

# Проверьте версию Python (должно быть 3.11.x или 3.12.x, НЕ 3.14!)
python --version
```

### Шаг 3: Настройка переменных окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

```env
# GigaChat API Configuration
GIGA_CREDENTIALS=your_base64_encoded_credentials_here

# Опционально: отключить проверку SSL сертификатов (для разработки)
# По умолчанию: true (проверка включена)
# Для отключения установите: false
GIGA_VERIFY_SSL=false

# OpenRouter API Configuration
OPENROUTER_API_KEY=your_openrouter_api_key_here
# Модель берется из профиля API, указывать не нужно
```

**Получение ключей:**

- **GigaChat**: Зарегистрируйтесь на https://developers.sber.ru/gigachat и получите credentials (обычно в формате base64)

**Важно:** Файл `.env` уже добавлен в `.gitignore` и не будет загружен в репозиторий.

### Шаг 4: Установите зависимости фронтенда

```bash
npm install
```

### Шаг 5: Запустите бэкенд сервер

В первом терминале (в корне проекта):
```bash
# Убедитесь, что виртуальное окружение активировано
uvicorn main:app --reload --port 8000
```

Вы должны увидеть: `Uvicorn running on http://127.0.0.1:8000`

⚠️ **НЕ открывайте `http://localhost:8000` в браузере** - это только API сервер!

### Шаг 6: Запустите фронтенд

Во втором терминале (в корне проекта):
```bash
npm run dev
```

Вы должны увидеть: `Local: http://localhost:5173/`

**🔴 КРИТИЧЕСКИ ВАЖНО: Откройте в браузере ТОЛЬКО этот URL:**

## Использование

1. Нажмите на кнопку "Выберите файл (CSV, xlsx, xls или PDF)"
2. Выберите файл формата .csv, .xlsx, .xls или .pdf
3. Данные автоматически загрузятся и отобразятся:
   - Информация о файле
   - Таблица с данными (первые 100 строк)
   - Базовая аналитика
   - Секция визуализации с возможностью создания графиков
   - Секция для нейросети (анализ AI)

### Создание графиков

В разделе "Визуализация данных":

1. Нажмите кнопку **"+ Добавить график"**
2. Выберите **тип графика** из выпадающего списка:
   - **Гистограмма (Bar)** - для сравнения категорий
   - **Линейный (Line)** - для временных рядов
   - **Круговая (Pie)** - для пропорций
   - **Точечная (Scatter)** - для корреляций
3. Выберите **столбец для X-оси** (или метки для круговой диаграммы)
4. Выберите **столбец для Y-оси** (опционально для круговой диаграммы)
5. Нажмите **"Создать график"**

Можно создать несколько графиков одновременно. Для удаления графика нажмите кнопку "Удалить" рядом с ним.

### AI-анализ данных

После загрузки файла автоматически запускаются два анализа данных:
1. **Анализ от GigaChat** - первый анализ от GigaChat
2. **Анализ от OpenRouter** - второй анализ от OpenRouter (появляется после завершения первого)

Оба анализа отображаются в отдельных секциях с одинаковым дизайном. Каждая секция имеет кнопку "📄 Скачать PDF" для сохранения анализа в PDF файл.

### Полноэкранный режим

Нажмите кнопку "⤢ Полный экран" в правом верхнем углу для перехода в полноэкранный режим. Для выхода используйте кнопку "⤓ Выйти из полноэкрана" или клавишу F11.

## Решение проблем

### Фронтенд не запускается

1. Убедитесь, что Node.js установлен: `node --version`
2. Удалите `node_modules` и `package-lock.json`, затем выполните `npm install` заново

### Бэкенд не запускается

1. Убедитесь, что виртуальное окружение активировано
2. Убедитесь, что установлены все зависимости: `pip install -r requirements.txt`
3. Проверьте версию Python: `python --version` (должна быть 3.8+)

### На localhost выводит "какую-то ерунду" (JSON код) или путаются порты

**Вы открыли неправильный URL!**

- ✅ **Откройте:** `http://localhost:5173` - это ваше приложение (фронтенд)
- ❌ **НЕ открывайте:** `http://localhost:8000` - это только API сервер (бэкенд)

Если вы видите JSON код или ошибку - значит вы открыли порт 8000. Закройте эту вкладку и откройте новую с адресом `http://localhost:5173`.

### Ошибки при анализе AI

1. Убедитесь, что файл `.env` создан в корне проекта
2. Проверьте, что `GIGA_CREDENTIALS` установлен в файле `.env`
3. Убедитесь, что библиотека `gigachat` установлена: `pip install gigachat`

### Ошибки с SSL сертификатами

Если возникают ошибки с SSL сертификатами, можно временно отключить проверку SSL (только для разработки):

Добавьте в файл `.env`:
```env
GIGA_VERIFY_SSL=false
```

**Важно:** Отключение проверки SSL снижает безопасность. Используйте только для разработки. Для продакшена рекомендуется настроить сертификаты правильно.

## Расширение функциональности

Для добавления нового AI-агента:

1. Создайте новый класс, наследующий `BaseAIAgent` в `ai_service.py`
2. Реализуйте метод `send_request()`
3. Добавьте поддержку в функцию `get_agent()`
4. Обновите документацию
