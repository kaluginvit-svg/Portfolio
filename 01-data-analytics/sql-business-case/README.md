# SQL: бизнес-кейс «Аналитика аптечных заказов»

Портфолио-проект по SQL: анализ данных заказов аптек и клиентов с использованием агрегации, оконных функций, соединений таблиц и подзапросов.

---

## О проекте

В проекте разобраны **8 аналитических запросов** к двум таблицам: заказы аптек (`pharma_orders`) и справочник клиентов (`customers`). Каждый запрос решает отдельную бизнес-задачу и демонстрирует определённые конструкции SQL.

**Цель:** показать владение базовыми и продвинутыми приёмами SQL в контексте реального кейса.

## Запросы и используемые конструкции SQL

### 1. Топ-3 аптеки по объёму продаж

**Файлы:** `sql/01_top3_pharmacies.sql`, `results/01_top3_pharmacies.txt`

**Задача:** вывести три аптеки с наибольшей выручкой (сумма `price * count` по каждой аптеке).

**Что делаем в запросе:**
- Считаем выручку по каждой аптеке: `SUM(price * count)`.
- Группируем по `pharmacy_name` (`GROUP BY`).
- Сортируем по убыванию выручки (`ORDER BY ... DESC`).
- Ограничиваем вывод тремя строками (`LIMIT 3`).

**Используемые конструкции:** `GROUP BY`, `SUM`, `ORDER BY`, `LIMIT`.

**Пример результата:** Cтоличная, Доктор Айболит, Здравсити — с указанием объёма продаж в рублях.

---

### 2. Топ-3 лекарства по объёму продаж

**Файлы:** `sql/02_top3_drugs.sql`, `results/02_top3_drugs.txt`

**Задача:** вывести три препарата с наибольшей выручкой.

**Что делаем в запросе:**
- Считаем выручку по каждому препарату: `SUM(price * count)`.
- Группируем по `drug` (`GROUP BY`).
- Сортируем по убыванию выручки и берём 3 строки (`ORDER BY ... LIMIT 3`).

**Используемые конструкции:** `GROUP BY`, `SUM`, `ORDER BY`, `LIMIT`.

**Пример результата:** Ибупрофен, Цитрамон, Аквафорте — с объёмом продаж в рублях.

---

### 3. Аптеки с оборотом более 1,8 млн рублей

**Файлы:** `sql/03_pharmacies_over_18m.sql`, `results/03_pharmacies_over_18m.txt`

**Задача:** вывести только те аптеки, у которых суммарный оборот (выручка) больше 1 800 000 руб.

**Что делаем в запросе:**
- Считаем оборот по аптеке: `SUM(price * count)` с `GROUP BY pharmacy_name`.
- Фильтруем по агрегату: `HAVING SUM(price * count) > 1800000` (не `WHERE`, так как условие по сумме).
- Сортируем по убыванию оборота.

**Используемые конструкции:** `GROUP BY`, `SUM`, `HAVING`, `ORDER BY`.

**Пример результата:** аптеки Cтоличная, Доктор Айболит, Здравсити с оборотами выше 1,8 млн.

---

### 4. Накопленная сумма продаж по каждой аптеке

**Файлы:** `sql/04_cumulative_sales_by_pharmacy.sql`, `results/04_cumulative_sales_by_pharmacy.txt`

**Задача:** по каждой строке заказа показать аптеку, дату, сумму заказа и **накопленную** сумму продаж внутри этой аптеки (по дате и order_id).

**Что делаем в запросе:**
- Для каждой строки считаем сумму заказа: `(price * count)`.
- Оконная функция: `SUM(price * count) OVER (PARTITION BY pharmacy_name ORDER BY report_date, order_id)` — накопленная сумма в рамках одной аптеки с порядком по дате и ID заказа.
- Сортируем вывод по аптеке, дате и заказу.

**Используемые конструкции:** оконная функция `SUM() OVER (PARTITION BY ... ORDER BY ...)`.

**Результат:** все строки заказов с полями аптека, дата, сумма заказа, накопленная сумма по аптеке (полный вывод в `results/`).

---

### 5. Количество клиентов по аптекам

**Файлы:** `sql/05_customers_per_pharmacy.sql`, `results/05_customers_per_pharmacy.txt`

**Задача:** по каждой аптеке посчитать **уникальное** количество клиентов, сделавших хотя бы один заказ.

**Что делаем в запросе:**
- Соединяем заказы и клиентов по `customer_id` (`JOIN`), чтобы опираться на реальных клиентов из справочника.
- Считаем уникальных клиентов: `COUNT(DISTINCT p.customer_id)` с группировкой по `pharmacy_name`.
- Сортируем аптеки по убыванию количества клиентов.

**Используемые конструкции:** `JOIN`, `COUNT(DISTINCT)`, `GROUP BY`, `ORDER BY`.

**Результат:** шесть аптек с количеством уникальных клиентов (от 426 до 441).

---

### 6. Лучшие клиенты (топ-10 по сумме заказов)

**Файлы:** `sql/06_best_customers.sql`, `results/06_best_customers.txt`

**Задача:** найти топ-10 клиентов по общей сумме заказов и пронумеровать их (1, 2, 3, …).

**Что делаем в запросе:**
- Соединяем заказы и клиентов (`JOIN`).
- Собираем ФИО: `first_name || ' ' || last_name || ' ' || COALESCE(second_name, '')`.
- Группируем по клиенту и считаем сумму заказов: `SUM(price * count)`.
- Ранжируем: `ROW_NUMBER() OVER (ORDER BY SUM(price * count) DESC)` в подзапросе.
- Во внешнем запросе оставляем только строки с номером места ≤ 10 (`WHERE место <= 10`).

**Используемые конструкции:** `JOIN`, `GROUP BY`, `SUM`, `ROW_NUMBER() OVER`, подзапрос, `LIMIT`/фильтр по месту.

**Результат:** 10 строк — customer_id, ФИО, сумма заказов (руб.), место в рейтинге.

---

### 7. Накопленная сумма по клиентам

**Файлы:** `sql/07_cumulative_sum_by_customer.sql`, `results/07_cumulative_sum_by_customer.txt`

**Задача:** по каждому заказу показать клиента (ФИО), дату, сумму заказа и **накопленную** сумму заказов этого клиента (по дате и order_id).

**Что делаем в запросе:**
- Соединяем заказы и клиентов (`JOIN`).
- Формируем одно поле ФИО конкатенацией имён и отчества.
- Считаем сумму заказа: `(price * count)`.
- Оконная функция: `SUM(price * count) OVER (PARTITION BY p.customer_id ORDER BY p.report_date, p.order_id)` — накопленная сумма по каждому клиенту.
- Сортируем по customer_id, дате и заказу.

**Используемые конструкции:** `JOIN`, конкатенация строк, `SUM() OVER (PARTITION BY ... ORDER BY ...)`.

**Результат:** все строки заказов с полями customer_id, ФИО, дата заказа, сумма заказа, накопленная сумма по клиенту (полный вывод в `results/`).

---

### 8. Самые частые клиенты аптек «Горздрав» и «Здравсити»

**Файлы:** `sql/08_top_customers_gorzdrav_zdravcity.sql`, `results/08_top_customers_gorzdrav_zdravcity.txt`

**Задача:** для аптек «Горздрав» и «Здравсити» отдельно вывести топ-10 клиентов по количеству заказов и объединить два списка в один результат.

**Что делаем в запросе:**
- Две CTE (временные таблицы через `WITH`):
  - по заказам «Горздрав»: JOIN с клиентами, `GROUP BY` клиент, считаем `COUNT(*)` заказов;
  - по заказам «Здравсити»: то же самое.
- В каждой CTE формируем ФИО и считаем число заказов.
- Для каждой сети отдельно считаем ранг: `ROW_NUMBER() OVER (ORDER BY кол_заказов DESC)` и оставляем топ-10 (`WHERE место <= 10` в подзапросе).
- Объединяем два набора строк с помощью `UNION ALL` (два списка по 10 строк — итого 20 строк с полями: аптека, ФИО, кол_заказов, место).
- Сортируем по названию аптеки и месту.

**Используемые конструкции:** `WITH` (CTE), `JOIN`, `GROUP BY`, `COUNT`, `ROW_NUMBER() OVER`, `UNION ALL`, подзапрос.

**Результат:** 20 строк — по 10 лучших клиентов для Горздрав и для Здравсити (аптека, ФИО, количество заказов, место).

## Используемые навыки SQL (кратко)

| Навык | Где используется |
|-------|-------------------|
| **WHERE** | Фильтрация (в т.ч. в CTE по названию аптеки) |
| **ORDER BY, LIMIT** | Сортировка и ограничение числа строк (топ-3, топ-10) |
| **GROUP BY** | Агрегация по аптеке, препарату, клиенту |
| **Агрегаты (SUM, COUNT)** | Выручка, оборот, число заказов, число клиентов |
| **HAVING** | Фильтр по обороту > 1,8 млн (запрос 3) |
| **CASE WHEN** | Не использовался в кейсе; владение — из курса |
| **JOIN** | Связка заказов и клиентов (запросы 5–8) |
| **UNION / UNION ALL** | Объединение топ-10 Горздрав и Здравсити (запрос 8) |
| **Подзапросы** | Ранжирование и отбор топ-10 (запросы 6, 8) |
| **WITH (CTE)** | Временные наборы для Горздрав и Здравсити (запрос 8) |
| **Функции дат/чисел/строк** | Конкатенация ФИО, COALESCE (запросы 6–8) |
| **Оконные функции** | SUM() OVER — накопленная сумма; ROW_NUMBER() — ранг (запросы 4, 6, 7, 8) |

## Как запустить

1. Клонировать репозиторий и перейти в папку проекта.
2. Убедиться, что установлен Python 3 и в проекте есть файлы `customers.csv` и `pharma_orders.csv`.
3. Выполнить:
   ```bash
   python run_queries.py
   ```
   Скрипт создаёт базу SQLite, загружает в неё CSV, выполняет все 8 запросов из папки `sql/` и записывает **полный** результат каждого запроса в соответствующий файл в `results/` (все строки, без ограничения по количеству).
4. Результаты просматривать в папке `results/` (формат: табуляция между столбцами, первая строка — заголовки).

При необходимости запросы можно выполнять вручную в любом клиенте SQLite (или перенести синтаксис в другой СУБД), подставив свои пути к таблицам.
