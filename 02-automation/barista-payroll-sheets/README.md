<a id="task"></a>

## 🎯 Задача

Вести зарплату в одной таблице: фиксировать смены с временем, собирать выручку по месяцам, считать часы и зарплату по сотрудникам — без постоянной правки диапазонов и копирования формул при каждом новом человеке или новом месяце.

<a id="solution"></a>

## ⚙️ Решение

Готовый скрипт на **Google Apps Script**, который:

- 🔍 находит колонки **по заголовкам**, а не по номерам — порядок столбцов можно менять;
- 📋 создаёт **кастомное меню** в таблице для настройки, обновления списков и формул;
- ▼ поддерживает **выпадающие списки** (сотрудники, время со шагом 30 минут);
- ∑ **проставляет и обновляет формулы** для часов, сумм по сменам, выручки и зарплаты;
- 📜 ведёт **журнал действий** на листе «Логи» (до 1000 записей);
- ✔ при первом запуске **проверяет структуру** таблицы и предупреждает о несоответствиях.

Итог для операционки: быстрее закрытие периода и предсказуемое поведение таблицы при росте данных.

<a id="quickstart"></a>

## 🚀 Быстрый старт

1. Откройте нужную книгу Google Sheets.
2. **Расширения → Apps Script** — вставьте код из `Автоматизация_зарплаты_бариста.gs`, сохраните.
3. Обновите страницу таблицы (F5).
4. **Автоматизация зарплаты → Настроить таблицу.**

Подробные шаги, типовые сценарии и troubleshooting — в `ИНСТРУКЦИЯ_по_использованию.md` и `ПРИМЕРЫ_ИСПОЛЬЗОВАНИЯ.md`.

<a id="structure"></a>

## 🗂️ Ожидаемая структура книги

| Лист | Колонки |
|------|---------|
| **Смены** | Дата, Сотрудник, Время начала, Время завершения, Часы работы (формула) |
| **Зарплата_период** | Сотрудник, Часы работы, Выручка, Зарплата |
| **Месяцы** *(декабрь, январь …)* | Дата, Сотрудник, Выручка |

Названия колонок должны совпадать с ожидаемыми (регистр не важен). Константы `SHEET_NAMES` и `COLUMN_NAMES` в начале скрипта можно подстроить под свою модель.

<a id="customize"></a>

## 🛠️ Настройка под себя

- **Ставка в формуле зарплаты** — функция `setupSalaryPeriodFormulas()` (пример в исходной документации).
- **Шаг времени** — `generateTimeOptions()` (по умолчанию 30 минут).
- **Новые поля** — расширение `COLUMN_NAMES` и соответствующей логики, затем «Обновить формулы».

## 🔒 Безопасность и риски
