# Примеры использования скрипта автоматизации зарплаты

## Сценарий 1: Первая настройка таблицы

### Шаг 1: Подготовка структуры
1. Создайте лист **"Смены"** с заголовками:
   - Дата | Сотрудник | Время начала | Время завершения | Часы работы

2. Создайте лист **"Зарплата_период"** с заголовками:
   - Сотрудник | Часы работы | Выручка | Зарплата

3. Создайте листы месяцев (например, **"декабрь"**, **"январь"**) с заголовками:
   - Дата | Сотрудник | Выручка

### Шаг 2: Запуск скрипта
1. Откройте **Расширения → Apps Script**
2. Вставьте код из файла `Автоматизация_зарплаты_бариста.gs`
3. Сохраните (Ctrl+S)
4. Вернитесь в таблицу и обновите страницу (F5)
5. Выберите **Автоматизация зарплаты → Настроить таблицу**

### Результат:
- ✅ Выпадающие списки настроены на листе "Смены"
- ✅ Формулы установлены на всех листах
- ✅ Создан лист "Логи" для отслеживания действий

---

## Сценарий 2: Добавление нового сотрудника

### Вариант А: Через меню
1. Выберите **Автоматизация зарплаты → Добавить нового сотрудника**
2. Введите имя: "Иван Петров"
3. Нажмите OK

### Результат:
- ✅ Новая строка добавлена в "Зарплата_период"
- ✅ Формулы автоматически применены
- ✅ Имя добавлено в выпадающие списки

### Вариант Б: Вручную
1. Откройте лист "Зарплата_период"
2. Добавьте новую строку внизу
3. Введите имя сотрудника в колонку "Сотрудник"
4. Формулы применятся автоматически (благодаря `onEdit`)

---

## Сценарий 3: Добавление нового месяца

1. Создайте новый лист с названием месяца (например, **"февраль"**)
2. Добавьте заголовки: Дата | Сотрудник | Выручка
3. Выберите **Автоматизация зарплаты → Обновить формулы**

### Результат:
- ✅ Новый лист автоматически подключен к расчетам
- ✅ Формулы на "Зарплата_период" обновлены для учета нового месяца

---

## Сценарий 4: Работа со сменами

1. Откройте лист **"Смены"**
2. В колонке **"Сотрудник"** выберите из выпадающего списка
3. В колонке **"Время начала"** выберите время (например, 09:00)
4. В колонке **"Время завершения"** выберите время (например, 17:00)
5. Колонка **"Часы работы"** заполнится автоматически

### Результат:
- ✅ Часы работы рассчитаны автоматически
- ✅ Данные автоматически суммируются в "Зарплата_период"

---

## Сценарий 5: Изменение ставки оплаты

Если у вас есть колонка "Ставка" в листе "Зарплата_период":

1. Добавьте колонку **"Ставка"** в "Зарплата_период"
2. Введите ставки для каждого сотрудника
3. Выберите **Автоматизация зарплаты → Обновить формулы**

### Результат:
- ✅ Формула зарплаты автоматически использует колонку "Ставка"
- ✅ Зарплата = Часы работы × Ставка

Если колонки "Ставка" нет, скрипт использует фиксированную ставку 500. Чтобы изменить её, отредактируйте скрипт:

```javascript
// В функции setupSalaryPeriodFormulas() найдите:
const formula = `=IF(${hoursCell}<>"", ${hoursCell} * 500, "")`;

// Измените 500 на нужную ставку:
const formula = `=IF(${hoursCell}<>"", ${hoursCell} * 600, "")`;
```

## Сценарий 7: Решение проблем

### Проблема: Формулы не работают
**Решение:**
1. Проверьте лист "Логи"
2. Выберите **Автоматизация зарплаты → Обновить формулы**
3. Убедитесь, что заголовки колонок точно совпадают

### Проблема: Выпадающие списки пустые
**Решение:**
1. Добавьте хотя бы одного сотрудника в таблицу
2. Выберите **Автоматизация зарплаты → Обновить выпадающие списки**

### Проблема: Новый сотрудник не появляется в списке
**Решение:**
1. Убедитесь, что имя сотрудника введено в колонку "Сотрудник"
2. Выберите **Автоматизация зарплаты → Обновить выпадающие списки**

## Сценарий 8: Масштабирование таблицы

### Добавление новых колонок:
1. Добавьте новую колонку (например, "Премия")
2. Добавьте название в константы `COLUMN_NAMES`:
   ```javascript
   BONUS: 'Премия'
   ```
3. Добавьте логику обработки в функции скрипта
4. Обновите формулы через меню

### Изменение структуры:
Скрипт автоматически адаптируется к:
- ✅ Изменению порядка колонок
- ✅ Добавлению новых колонок
- ✅ Удалению неиспользуемых колонок

Не работает при:
- ❌ Изменении названий используемых колонок
- ❌ Удалении используемых колонок

---

## Сценарий 9: Работа с большим объемом данных

Скрипт оптимизирован для работы с большими таблицами:
- Обрабатывает до 1000 строк за раз
- Использует пакетные операции
- Логирует прогресс выполнения

При работе с очень большими таблицами (>10000 строк):
1. Разбейте данные на периоды
2. Используйте фильтры для обработки частями
3. Проверяйте логи на наличие ошибок

---

## Сценарий 10: Резервное копирование

Перед первым запуском:
1. Создайте копию таблицы: **Файл → Создать копию**
2. Сохраните копию с названием "Резервная копия"
3. Работайте с оригиналом
4. При необходимости можно восстановить из копии

---

## Полезные советы:

1. **Регулярно проверяйте логи** - они помогут быстро найти проблемы
2. **Используйте меню** - все функции доступны через удобное меню
3. **Делайте резервные копии** - перед большими изменениями
4. **Проверяйте структуру** - скрипт предупредит о проблемах при инициализации
5. **Обновляйте формулы** - после изменения структуры таблицы
