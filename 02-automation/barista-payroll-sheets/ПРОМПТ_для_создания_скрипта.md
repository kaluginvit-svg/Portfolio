# Промпт для создания Apps Script для автоматизации таблицы "Зарплата_бариста_Декабрь_2025"

## Задача
Создать Apps Script для Google Sheets, который полностью автоматизирует таблицу расчета зарплаты бариста. Таблица должна быть масштабируемой, работать с динамическими данными и не зависеть от конкретных номеров строк/столбцов.

## Структура таблицы

### Листы:
1. **"Смены"** - содержит информацию о сменах сотрудников
   - Колонки: Дата, Сотрудник, Время начала, Время завершения, Часы работы, и т.д.
   - Требования: выпадающие списки для Сотрудника, Времени начала и завершения

2. **"декабрь", "январь", и т.д.** - листы по месяцам с выручкой
   - Содержат данные о выручке за соответствующий месяц
   - Формулы должны брать данные с других листов

3. **"Зарплата_период"** - итоговый расчет зарплаты
   - Автоматически собирает данные со всех листов
   - Должен работать при добавлении новых строк с сотрудниками

## Требования к скрипту

### 1. Принципы работы:
- **Не завязан на номера колонок** - использует поиск заголовков по названиям
- **Опирается на смысловые якоря** - заголовки, названия листов, параметры
- **Работает пакетно** - обрабатывает диапазоны данных целиком
- **Понятные сообщения** - логирует что сделал, сколько строк обработал, где проблема

### 2. Функциональность:

#### Настройка выпадающих списков на листе "Смены":
- Создать выпадающий список для колонки "Сотрудник" из списка всех сотрудников
- Создать выпадающие списки для времени начала и завершения (например, с шагом 30 минут)
- При добавлении нового сотрудника автоматически обновлять список

#### Автоматизация формул:
- Формулы должны использовать функции типа `VLOOKUP`, `SUMIF`, `INDEX/MATCH` с поиском по заголовкам
- Формулы должны быть привязаны к именам сотрудников, а не к конкретным ячейкам
- При добавлении нового сотрудника формулы должны автоматически применяться

#### Масштабируемость:
- При добавлении нового месяца (нового листа) автоматически подключать его к расчетам
- При добавлении нового сотрудника автоматически создавать строки в "Зарплата_период"
- При изменении периода автоматически пересчитывать данные

### 3. Функции скрипта:

```javascript
// Основные функции:
- setupDataValidation() - настройка выпадающих списков
- setupFormulas() - установка формул на всех листах
- updateEmployeeList() - обновление списка сотрудников
- calculateSalary() - расчет зарплаты за период
- addNewEmployee() - добавление нового сотрудника
- addNewMonth() - добавление нового месяца
- logActivity() - логирование действий
```

### 4. Алгоритм работы:

1. **Инициализация:**
   - Найти все листы с месяцами (по названиям)
   - Найти лист "Смены" и "Зарплата_период"
   - Определить структуру таблиц по заголовкам

2. **Настройка валидации:**
   - Извлечь список всех сотрудников из листа "Смены" или отдельного справочника
   - Создать выпадающие списки для соответствующих колонок
   - Настроить валидацию времени

3. **Установка формул:**
   - Для каждого листа месяца найти колонки по заголовкам
   - Установить формулы расчета выручки
   - На листе "Зарплата_период" установить формулы суммирования по сотрудникам

4. **Обработка новых данных:**
   - При добавлении строки в "Зарплата_период" автоматически копировать формулы
   - При добавлении нового месяца создавать связи с расчетами
   - При добавлении сотрудника обновлять все связанные формулы

### 5. Примеры формул (используя поиск по заголовкам):

```javascript
// Вместо: =SUM(B2:B10)
// Использовать: =SUM(INDEX(Смены!A:Z, 0, MATCH("Часы работы", Смены!1:1, 0)))

// Вместо: =VLOOKUP(A2, декабрь!A:B, 2, FALSE)
// Использовать: =INDEX(декабрь!A:Z, MATCH(A2, INDEX(декабрь!A:Z, 0, MATCH("Сотрудник", декабрь!1:1, 0)), 0), MATCH("Выручка", декабрь!1:1, 0))
```

### 6. Логирование:
- Каждая функция должна логировать начало и завершение работы
- Указывать количество обработанных строк/ячеек
- Выводить ошибки с указанием конкретного места проблемы

### 7. Обработка ошибок:
- Проверка существования листов
- Проверка наличия необходимых заголовков
- Валидация данных перед применением формул
- Понятные сообщения об ошибках

## Результат
Скрипт должен быть готов к использованию, полностью документирован и легко расширяемым для будущих изменений структуры таблицы.
