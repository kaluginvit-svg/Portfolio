# ИИ-агент: разработка SEO-ядра

> **Проблема:** ручной сбор SEO-ядра по списку URL занимает дни — копипаст, прогон через инструменты, сведение результатов. Нужен агент, который пройдёт по N страницам и соберёт **одно совокупное** ядро.  
> **Решение:** FastAPI-бэкенд + минимальный фронт + Docker Compose. Агент через ProxyAPI (OpenAI-совместимый) формирует план, обходит каждый URL, собирает ответы, объединяет в одно SEO-ядро. Прогресс приходит на фронт по SSE.  
> **Стек:** FastAPI, OpenAI / ProxyAPI, Docker Compose, ванильный JS, Server-Sent Events.  
> **Ценность:** за один заход — объединённое SEO-ядро по списку страниц с потоковым логом анализа; шаблон того, как обернуть LLM-агента в продуктовый веб-интерфейс.

---

Генерация совокупного SEO-ядра по списку URL с помощью **облачного API** (ProxyAPI / OpenAI).

## Архитектура

- **ИИ-агент**: облачный OpenAI-совместимый API (например ProxyAPI: https://api.proxyapi.ru/openai/v1).
- **Бэкенд**: FastAPI, папка `backend/` — вызывает API по `OPENAI_BASE_URL`.
- **Фронтенд**: папка `frontend/`, отдаётся бэкендом.
- **Docker Compose**: бэкенд в контейнере.

## Требования

1. Ключ API (ProxyAPI или OpenAI) в `.env`: `OPENAI_API_KEY=...`
2. В `.env` указан `OPENAI_BASE_URL` (например `https://api.proxyapi.ru/openai/v1`).

## Запуск

1. Скопируйте конфиг и задайте ключ и URL API:
   ```bash
   copy .env.example .env
   ```
   В `.env` укажите `OPENAI_API_KEY`, `OPENAI_BASE_URL`, при необходимости `OPENAI_MODEL` (по умолчанию gpt-4o-mini).
2. Запустите проект:
   - Дважды щёлкните **`start.bat`** или в PowerShell: `.\start.bat`
   - Или: `docker compose -p seo-agent up -d`
3. Откройте в браузере: **http://localhost:8000**

## Функционал

- В поле ввода вставьте **список URL** (каждый с новой строки или через запятую).
- Бэкенд запрашивает у ИИ план анализа, обходит каждый URL, собирает ответы и формирует **одно совокупное SEO-ядро**.
- Лог и результат отображаются в реальном времени (SSE).

## Переменные окружения

| Переменная           | Описание |
|----------------------|----------|
| `OPENAI_BASE_URL`    | URL API (по умолчанию https://api.proxyapi.ru/openai/v1) |
| `OPENAI_MODEL`       | Модель (по умолчанию gpt-4o-mini) |
| `OPENAI_API_KEY`     | Ключ API (обязательно для облака) |
| `USE_URL_DIRECTLY`   | `true` — передавать в промпты только URL; `false` — подставлять HTML страницы |

## Устранение неполадок

**Ошибка соединения с API**  
- Проверьте `OPENAI_BASE_URL` и доступность API из сети.  
- Проверьте баланс в личном кабинете ProxyAPI (если используете ProxyAPI).  
- После изменения `.env` перезапустите контейнеры: `docker compose -p seo-agent down` и `docker compose -p seo-agent up -d`.

**Ошибка 404 при нажатии «Анализировать»**  
- Открывайте приложение по адресу http://localhost:8000 (не с диска через `file://`).  
- Проверьте: http://localhost:8000/api/status — должна открыться страница с конфигом.

**Перезапуск контейнера**  
```bash
docker compose -p seo-agent down
docker compose -p seo-agent up -d
```
