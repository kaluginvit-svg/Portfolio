# RAG

RAG (Retrieval-Augmented Generation) — подход, в котором языковая модель отвечает не только по своей внутренней памяти, но и с опорой на найденные внешние документы.

Обычно пайплайн состоит из трех шагов:
1. Вопрос пользователя преобразуется в эмбеддинг.
2. По эмбеддингу ищутся релевантные фрагменты в векторной базе, например в Pinecone.
3. Найденные фрагменты добавляются в prompt, после чего модель формирует итоговый ответ.

Преимущество RAG в том, что можно обновлять базу знаний без дообучения модели и снижать вероятность галлюцинаций.
