Ты — senior Python-разработчик и архитектор MCP-систем. Создай полный рабочий проект MCP SERVER под названием product-mcp.

Цель проекта:
построить базовый финансовый MCP-сервер, который предоставляет стандартизированные инструменты для работы с финансовыми данными компании. Этот сервер должен быть именно MCP-слоем, а не агентом и не оркестратором. Он должен уметь хранить, импортировать, выдавать и рассчитывать финансовые данные через MCP tools. Позже на его основе будет построен orchestration-agent, который будет вызывать эти инструменты.

========================================
1. ОБЩАЯ ИДЕЯ
========================================

Сделай product-mcp как локальный MCP-сервер для финансового и корпоративного использования.

Он должен:
- работать локально
- использовать SQLite3
- запускаться командой python server.py
- экспортировать tools в MCP-совместимом формате
- иметь JSON schema для каждого инструмента
- быть независимым от внешних API
- быть расширяемым для будущего orchestration-layer

На этом этапе НЕ нужно строить AI-агента, reasoning-движок или orchestration workflow.
Нужен именно надежный MCP server with tools.

========================================
2. ГРАНИЦЫ MVP
========================================

В первой версии сервер должен покрывать 4 класса задач:

1) хранение и импорт финансовых данных
2) выборка и фильтрация данных
3) базовые финансовые расчеты
4) стандартные tool-интерфейсы для будущего оркестратора

Сервер не должен сам “думать”, инициировать workflow или строить сложные автономные сценарии.
Он только:
- принимает запрос
- валидирует параметры
- читает/пишет в БД
- считает
- возвращает структурированный результат

========================================
4. ИНИЦИАЛИЗАЦИЯ
========================================

При первом запуске сервер должен:
- автоматически создавать базу данных
- автоматически создавать все таблицы
- автоматически создавать папки data/imports и data/exports
- автоматически загружать seed-данные

Seed-данные должны включать:
- 1–2 компании
- 12 месяцев P&L
- 12 месяцев cash flow
- 12 месяцев balance
- AR/AP записи
- payment records
- cash positions
- 5–10 contracts
- 3–5 investment projects

Seed должен быть реалистичным и демонстрировать:
- выручку
- OPEX
- EBITDA
- AR/AP
- платежный календарь
- остатки денежных средств
- базовую инвестиционную оценку

========================================
5. MCP TOOLS
========================================

Реализуй MCP tools как основной интерфейс сервера.

Каждый tool должен иметь:
- name
- description
- input schema
- output schema
- обработку ошибок
- предсказуемый JSON-ответ

Сделай следующие инструменты:

1. health_check()
Возвращает:
- server_status
- db_path
- available_tools
- counts_by_table

2. list_companies()
Возвращает список компаний.

3. import_csv(file_path: string, statement_type: string, company_name?: string, version?: string)
Импортирует CSV-файл.
Поддержать statement_type:
- pnl
- cashflow
- balance
- ap
- ar
- payments
- kpi
- budget

Функция должна:
- читать CSV
- распознавать колонки
- маппить типовые заголовки
- валидировать строки
- записывать в financial_records или budget_records
- возвращать imported_count, rejected_count, errors

4. import_contract(file_path: string)
Импортирует TXT или PDF договор.
Функция должна:
- извлечь текст
- сохранить raw_text
- попробовать достать contract_name, counterparty, start_date, end_date, payment_terms, penalty_terms, amount, currency
- записать в contracts
- вернуть parsed fields и warnings

5. list_financial_records(
   statement_type?: string,
   company_name?: string,
   start_date?: string,
   end_date?: string,
   category?: string,
   department?: string,
   project?: string,
   counterparty?: string
)
Возвращает записи financial_records по фильтрам.

6. list_budget_records(
   company_name?: string,
   version?: string,
   start_date?: string,
   end_date?: string,
   category?: string
)
Возвращает budget_records по фильтрам.

7. list_cash_positions(company_name?: string, position_date?: string)
Возвращает денежные остатки.

8. list_contracts(active_only?: bool = false)
Возвращает список договоров.

9. list_investment_projects()
Возвращает список проектов.

10. calculate_kpis(period_start?: string, period_end?: string, company_name?: string)
Считает и возвращает:
- total_revenue
- total_opex
- gross_profit
- EBITDA
- EBITDA_margin
- net_cash_flow
- accounts_receivable_total
- accounts_payable_total
- cash_balance

11. plan_vs_fact(period_start?: string, period_end?: string, company_name?: string)
Сравнивает budget_records и financial_records.
Возвращает:
- total_plan
- total_fact
- variance_abs
- variance_pct
- breakdown_by_category

12. liquidity_forecast(days: integer = 90, company_name?: string)
Строит простой прогноз ликвидности:
- opening_cash
- projected_inflows
- projected_outflows
- ending_cash
- daily_projection
- risk_flags

13. payment_calendar(start_date?: string, end_date?: string, company_name?: string)
Возвращает календарь платежей и поступлений.
Добавить:
- direction: inflow / outflow
- overdue: true / false

14. contract_risk_scan()
Сканирует contracts и возвращает риски:
- expiring_soon
- missing_payment_terms
- missing_penalty_terms
- missing_amount
- no_end_date
- penalty_detected
Сохранять риски в alerts.

15. evaluate_investment(project_id: integer)
Считает:
- NPV
- IRR
- payback_period
- profitability_index
- simple recommendation

16. add_investment_project(
   project_name: string,
   company_name: string,
   initial_investment: float,
   discount_rate: float,
   hurdle_rate: float,
   scenario_json: string,
   notes?: string
)
Добавляет инвестиционный проект.

17. find_records(query: string)
Поиск по category, subcategory, counterparty, project, contract_name, source_file.

18. export_report(report_type: string, output_format?: string, output_path?: string)
Поддержать report_type:
- kpis
- plan_vs_fact
- liquidity_forecast
- payment_calendar
- contract_risks
- investment_evaluation

Сохранять файл в data/exports/.

19. calculate(expression: string)
Безопасный калькулятор.
eval запрещен.
Использовать AST.
Разрешить:
- + - * / ** % ()
- min max round abs
