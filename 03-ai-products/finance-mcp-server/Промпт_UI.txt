Ты — senior frontend/fullstack разработчик. Создай полный рабочий UI-проект для внутреннего использования поверх уже существующего MCP server под названием product-mcp.

Цель проекта:
сделать удобный внутренний web-интерфейс для финансового MCP, чтобы финансист, казначей, CFO или аналитик могли:
- загружать XLSX / CSV / PDF файлы
- запускать MCP tools через UI
- просматривать результаты в таблицах и карточках
- видеть ошибки импорта
- смотреть KPI, plan-fact, ликвидность, платежный календарь, риски по договорам и инвестиционные проекты
- работать без прямого обращения к MCP через чат или API вручную

Важно:
это НЕ orchestration-agent.
Это НЕ AI-чат.
Это UI-клиент поверх MCP backend.

========================================
1. ОБЩАЯ АРХИТЕКТУРА
========================================

Создай web-приложение с двумя слоями:

1) Frontend UI
2) Тонкий backend adapter, который вызывает MCP server tools

Предположения:
- MCP server product-mcp уже существует
- MCP server предоставляет tools и JSON schema
- UI не должен содержать бизнес-логику расчетов
- бизнес-логика остается в MCP
- UI только собирает ввод, валидирует базовые поля, вызывает tools и показывает ответ

Сделай проект production-like MVP, который можно запустить локально.

========================================
2. СТЕК
========================================

Используй современный и практичный стек.

Предпочтительный вариант:
- Next.js 14+ или React + Vite
- TypeScript
- Tailwind CSS
- shadcn/ui или аккуратный компонентный набор
- React Query / TanStack Query
- Zod для валидации
- Node.js backend routes или легкий express/fastify adapter layer

Если решишь использовать Next.js:
- app router
- server actions или route handlers для вызова MCP adapter
- API routes для file upload и tool execution

========================================
3. ОСНОВНАЯ ИДЕЯ UX
========================================

UI должен быть не “чисто техническим”, а ориентированным на финансового пользователя.

Нужен clean internal dashboard style:
- левое меню
- верхняя панель
- страницы с таблицами, фильтрами, карточками, alert-блоками
- upload forms
- action buttons для запуска MCP tools
- понятные состояния loading / success / error / empty state

Пользователь должен иметь возможность:
- открыть раздел
- загрузить файл
- выбрать тип данных
- увидеть результат импорта
- запустить расчет
- посмотреть ответ MCP в удобном виде
- скачать экспортированный отчет

========================================
5. MCP ADAPTER LAYER
========================================

Создай тонкий backend adapter между UI и MCP server.

Нужно:
- централизованный MCP client module
- универсальный вызов tool по name + payload
- обработка ошибок
- логирование запросов
- нормализация ответов

Сделай модуль примерно такого назначения:
- mcpClient.callTool(toolName, payload)
- getAvailableTools()
- uploadAndImportFile(...)
- handleMcpError(...)

Если MCP server не предоставляет готовый transport:
- сделай абстракцию клиента через HTTP/JSON
- заложи структуру, чтобы позже можно было заменить transport без переписывания UI
