# SQL Игра - Обучалка по SQL запросам

## 🎮 Описание

Интерактивная игра для изучения SQL, где система дает задания разной сложности, а пользователь вводит SQL запросы. Система проверяет правильность запросов и начисляет очки.

## 📋 Возможности

- ✅ 5 уровней сложности (от базовых SELECT до сложных JOIN)
- ✅ 15+ заданий с разными категориями
- ✅ Проверка правильности SQL запросов
- ✅ Система подсказок
- ✅ Подсчет очков и статистика
- ✅ Сохранение результатов

## 🚀 Установка и запуск

### 1. Подготовка базы данных

Сначала создайте основную базу данных с товарами и заказами:

```bash
# Создание таблиц
python create_tables_python.py

# Заполнение данными
python fill_products_python.py
```

Или используйте SQL файлы в DBeaver:
- `create_tables.sql` - создание таблиц
- `setup_products_orders.sql` - создание и заполнение

### 2. Запуск игры

```bash
python sql_game.py
```

При первом запуске игра автоматически создаст базу данных `sql_game.db` с заданиями.

## 🎯 Уровни сложности

### Уровень 1: Базовые SELECT
- Выборка всех данных
- Выборка конкретных столбцов
- Подсчет записей

### Уровень 2: WHERE и фильтрация
- Условия сравнения
- Поиск по категориям
- LIKE для поиска по шаблону

### Уровень 3: Сортировка и ограничение
- ORDER BY
- LIMIT
- BETWEEN

### Уровень 4: Агрегация
- GROUP BY
- HAVING
- Агрегатные функции (COUNT, AVG, SUM, MIN, MAX)

### Уровень 5: JOIN
- INNER JOIN
- Множественные JOIN
- JOIN с агрегацией

## 📝 Примеры заданий

### Уровень 1: "Выбери все товары"
```
Описание: Напиши запрос, который выберет все товары из таблицы products
Правильный ответ: SELECT * FROM products;
```

### Уровень 2: "Товары дороже 10000"
```
Описание: Найди все товары, цена которых больше 10000 рублей
Правильный ответ: SELECT * FROM products WHERE price > 10000;
```

### Уровень 4: "Средняя цена по категориям"
```
Описание: Посчитай среднюю цену товаров в каждой категории
Правильный ответ: SELECT category, AVG(price) FROM products GROUP BY category;
```

### Уровень 5: "Товары в заказах"
```
Описание: Выбери все товары, которые есть в заказах. Используй JOIN
Правильный ответ: SELECT p.name, p.category, oi.quantity 
                  FROM products p 
                  INNER JOIN order_items oi ON p.id = oi.product_id;
```

## 🎮 Как играть

1. **Запустите игру**: `python sql_game.py`
2. **Введите имя** для сохранения статистики
3. **Выберите уровень** сложности
4. **Читайте задание** и вводите SQL запрос
5. **Получайте обратную связь** - система покажет, правильный ли запрос
6. **Используйте подсказки** если застряли
7. **Зарабатывайте очки** за правильные ответы

## 📊 Система очков

- **Базовые очки**: зависят от сложности задания (10-45 очков)
- **Штраф за повторные попытки**: -20% очков за каждую дополнительную попытку
- **Максимум попыток**: 3 на задание

## 🔍 Проверка запросов

Система проверяет:
- ✅ Правильность синтаксиса SQL
- ✅ Наличие требуемых столбцов
- ✅ Использование нужных операторов (WHERE, JOIN, GROUP BY и т.д.)
- ✅ Количество строк (если указано в задании)

## 🐛 Решение проблем

### Ошибка: "База данных не найдена"
Убедитесь, что создали `products_orders.db` перед запуском игры.

### Ошибка: "SQL error"
Проверьте синтаксис запроса. Система покажет детали ошибки.

### Запрос правильный, но не принимается
Система проверяет столбцы и структуру. Убедитесь, что:
- Выбраны правильные столбцы
- Использованы нужные операторы (WHERE, JOIN и т.д.)
- Запрос выполняется без ошибок
