## Обзор

Этот документ объясняет, как создать SQL-запрос для вставки 1000 случайных записей о товарах в базу данных SQLite. Мы используем **рекурсивный CTE (Common Table Expression)** для генерации множества записей и функции SQLite для создания случайных данных.

### Что такое INSERT?

**INSERT** - это SQL-команда для добавления новых строк (записей) в таблицу базы данных.

**Базовый синтаксис:**
```sql
INSERT INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...);
```

## Создание таблицы

Перед вставкой данных нужно создать таблицу. В нашем случае таблица `products` содержит следующие поля:

```sql
CREATE TABLE IF NOT EXISTS products (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  -- Автоматически увеличивающийся ID
    article TEXT NOT NULL UNIQUE,          -- Уникальный артикул товара
    name TEXT NOT NULL,                    -- Название товара
    category TEXT NOT NULL,                 -- Категория товара
    description TEXT,                       -- Описание товара
    price REAL NOT NULL CHECK(price > 0),  -- Цена (должна быть больше 0)
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP  -- Дата создания
);
```

### Объяснение полей:

- **`id`** - Первичный ключ, автоматически увеличивается при каждой новой записи
- **`article`** - Артикул товара, должен быть уникальным (`UNIQUE`)
- **`name`** - Название товара, обязательное поле (`NOT NULL`)
- **`category`** - Категория товара, обязательное поле
- **`description`** - Описание товара, может быть пустым
- **`price`** - Цена товара, должна быть больше 0 (`CHECK(price > 0)`)
- **`created_at`** - Дата создания, автоматически заполняется текущей датой и временем

## Рекурсивный CTE

**CTE (Common Table Expression)** - это временный именованный набор результатов, который существует только в рамках выполнения одного SQL-запроса.

**Рекурсивный CTE** позволяет нам генерировать последовательность чисел от 1 до 1000:

```sql
WITH RECURSIVE generate_products(n) AS (
    -- Базовый случай: начинаем с 1
    SELECT 1
    
    UNION ALL
    
    -- Рекурсивный случай: увеличиваем n до 1000
    SELECT n + 1
    FROM generate_products
    WHERE n < 1000
)
SELECT ... FROM generate_products;
```

### Как это работает:

1. **`WITH RECURSIVE`** - объявляет рекурсивное CTE с именем `generate_products`
2. **`generate_products(n)`** - создает временную таблицу с одним столбцом `n`
3. **Базовый случай** (`SELECT 1`) - начальное значение, с которого начинается рекурсия
4. **`UNION ALL`** - объединяет результаты базового и рекурсивного случаев
5. **Рекурсивный случай** (`SELECT n + 1 FROM generate_products WHERE n < 1000`) - увеличивает `n` на 1, пока не достигнет 1000
6. **`WHERE n < 1000`** - условие остановки рекурсии

### Визуализация работы:

```
Итерация 1: n = 1
Итерация 2: n = 2
Итерация 3: n = 3
...
Итерация 1000: n = 1000
Итерация 1001: условие n < 1000 не выполняется → остановка
```

## Генерация случайных данных

Для генерации случайных данных в SQLite используется функция **`random()`**, которая возвращает случайное целое число.

### 1. Генерация артикула

```sql
'ART-' || printf('%05d', abs(random() % 100000)) || '-' || n AS article
```

**Разбор по частям:**

- **`random()`** - генерирует случайное целое число (может быть отрицательным)
- **`abs(random() % 100000)`** - берет остаток от деления на 100000 и делает его положительным
  - `%` - оператор остатка от деления (модуло)
  - `abs()` - функция абсолютного значения
  - Результат: число от 0 до 99999
- **`printf('%05d', ...)`** - форматирует число как строку с 5 цифрами, добавляя нули слева
  - `%05d` означает: целое число (d), минимум 5 символов, заполнить нулями слева
  - Пример: `123` → `00123`
- **`||`** - оператор конкатенации (объединения) строк
- **Результат:** `ART-00123-1`, `ART-45678-2` и т.д.

### 2. Генерация названия товара

```sql
CASE (abs(random() % 10))
    WHEN 0 THEN 'Смартфон ' || (abs(random() % 50) + 1)
    WHEN 1 THEN 'Ноутбук ' || (abs(random() % 30) + 1)
    ...
    ELSE 'Планшет ' || (abs(random() % 22) + 1)
END AS name
```

**Разбор:**

- **`CASE ... WHEN ... THEN ... ELSE ... END`** - условный оператор (аналог if-else)
- **`abs(random() % 10)`** - случайное число от 0 до 9
- **`WHEN 0 THEN`** - если результат равен 0, выбираем первый вариант
- **`abs(random() % 50) + 1`** - случайное число от 1 до 50
  - `% 50` дает число от 0 до 49
  - `+ 1` сдвигает диапазон на 1-50
- **Результат:** `Смартфон 25`, `Ноутбук 12`, `Планшет 7` и т.д.

### 3. Генерация категории

```sql
CASE (abs(random() % 5))
    WHEN 0 THEN 'Электроника'
    WHEN 1 THEN 'Компьютеры'
    WHEN 2 THEN 'Аксессуары'
    WHEN 3 THEN 'Периферия'
    ELSE 'Гаджеты'
END AS category
```

**Разбор:**

- **`abs(random() % 5)`** - случайное число от 0 до 4
- В зависимости от результата выбирается одна из 5 категорий
- **Результат:** равномерное распределение по категориям

### 4. Генерация описания

```sql
'Описание товара #' || n || '. Качественный продукт... ' ||
CASE (abs(random() % 3))
    WHEN 0 THEN 'Рекомендуется для профессионального использования.'
    WHEN 1 THEN 'Идеально подходит для дома и офиса.'
    ELSE 'Проверенное качество и надежность.'
END AS description
```

**Разбор:**

- **`'Описание товара #' || n`** - добавляет номер товара в описание
- **`||`** - объединяет строки
- **`CASE ... END`** - добавляет случайное окончание описания
- **Результат:** `Описание товара #1. Качественный продукт... Идеально подходит для дома и офиса.`

### 5. Генерация цены

```sql
round((abs(random() % 49500) + 500) + (abs(random() % 100) / 100.0), 2) AS price
```

**Разбор по частям:**

- **`abs(random() % 49500)`** - случайное число от 0 до 49499
- **`+ 500`** - сдвигает диапазон на 500-49999
- **`abs(random() % 100) / 100.0`** - добавляет копейки (0.00 - 0.99)
  - `% 100` дает число от 0 до 99
  - `/ 100.0` преобразует в десятичную дробь
- **`round(..., 2)`** - округляет до 2 знаков после запятой
- **Результат:** цена от 500.00 до 50000.99 рублей

## Функции SQLite

### Основные функции, используемые в запросе:

| Функция | Описание | Пример |
|---------|----------|--------|
| `random()` | Возвращает случайное целое число | `random()` → `-1234567890` |
| `abs(x)` | Возвращает абсолютное значение | `abs(-5)` → `5` |
| `printf(format, ...)` | Форматирует строку | `printf('%05d', 123)` → `00123` |
| `round(x, n)` | Округляет число до n знаков | `round(3.14159, 2)` → `3.14` |
| `||` | Конкатенация строк | `'Hello' \|\| ' World'` → `'Hello World'` |
| `%` | Остаток от деления | `10 % 3` → `1` |

## Полный пример

### Структура запроса:

```sql
INSERT INTO products (article, name, category, description, price)
WITH RECURSIVE generate_products(n) AS (
    SELECT 1
    UNION ALL
    SELECT n + 1
    FROM generate_products
    WHERE n < 1000
)
SELECT 
    -- Генерация артикула
    'ART-' || printf('%05d', abs(random() % 100000)) || '-' || n AS article,
    
    -- Генерация названия
    CASE (abs(random() % 10))
        WHEN 0 THEN 'Смартфон ' || (abs(random() % 50) + 1)
        ...
    END AS name,
    
    -- Генерация категории
    CASE (abs(random() % 5))
        WHEN 0 THEN 'Электроника'
        ...
    END AS category,
    
    -- Генерация описания
    'Описание товара #' || n || '...' AS description,
    
    -- Генерация цены
    round((abs(random() % 49500) + 500) + (abs(random() % 100) / 100.0), 2) AS price

FROM generate_products;
```

### Пошаговое выполнение:

1. **CTE генерирует числа от 1 до 1000**
2. **Для каждого числа выполняется SELECT**, который:
   - Генерирует артикул
   - Генерирует название товара
   - Генерирует категорию
   - Генерирует описание
   - Генерирует цену
3. **Результаты SELECT вставляются в таблицу products**

### Пример результата:

| id | article | name | category | description | price |
|----|---------|------|----------|-------------|-------|
| 1 | ART-12345-1 | Смартфон 25 | Электроника | Описание товара #1... | 12500.50 |
| 2 | ART-67890-2 | Ноутбук 12 | Компьютеры | Описание товара #2... | 35000.75 |
| 3 | ART-11111-3 | Планшет 7 | Гаджеты | Описание товара #3... | 8500.00 |

## Проверка результата

После выполнения INSERT можно проверить результат:

```sql
-- Количество записей
SELECT COUNT(*) AS total_products FROM products;

-- Статистика по ценам
SELECT 
    MIN(price) AS min_price,
    MAX(price) AS max_price,
    round(AVG(price), 2) AS avg_price
FROM products;

-- Количество товаров по категориям
SELECT category, COUNT(*) AS count
FROM products
GROUP BY category;

-- Показать первые 10 записей
SELECT * FROM products LIMIT 10;
```

## Важные замечания

1. **Производительность**: Вставка 1000 записей может занять несколько секунд
2. **Уникальность артикулов**: Используется `UNIQUE`, поэтому дубликаты не допускаются
3. **Случайность**: Функция `random()` генерирует разные значения при каждом запуске
4. **Транзакции**: Для больших объемов данных рекомендуется использовать транзакции

### Использование транзакций:

```sql
BEGIN TRANSACTION;

INSERT INTO products (...) VALUES (...);
-- много записей

COMMIT;
```

Это ускорит выполнение и обеспечит целостность данных.
