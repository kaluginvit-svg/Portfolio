## Обзор оператора LIKE

**LIKE** - это SQL-оператор для поиска строк по шаблону (паттерну). Он используется в условии WHERE для фильтрации данных на основе частичного совпадения текста.

### Когда используется LIKE?

- Поиск товаров по части названия
- Фильтрация по артикулам с определенным форматом
- Поиск по описанию товара
- Поиск email-адресов или других текстовых данных
- Валидация формата данных

### ⚠️ Важно!

**LIKE работает только с текстовыми типами данных** (TEXT, VARCHAR, CHAR). Для числовых типов нужно сначала преобразовать в текст.

---

## Базовый синтаксис

```sql
SELECT column1, column2, ...
FROM table_name
WHERE column LIKE pattern;
```

### Разбор синтаксиса:

- **`LIKE`** - ключевое слово оператора
- **`pattern`** - шаблон для поиска (может содержать специальные символы)
- Шаблон обычно заключается в одинарные кавычки: `'pattern'`

### Простой пример:

```sql
-- Найти товары, в названии которых есть "Смартфон"
SELECT * FROM products 
WHERE name LIKE '%Смартфон%';
```

---

## Символы подстановки

LIKE использует два специальных символа для создания шаблонов:

| Символ | Описание | Пример |
|--------|----------|--------|
| `%` | Любое количество любых символов (0 или более) | `'%фон'` найдет "телефон", "смартфон", "фон" |
| `_` | Ровно один любой символ | `'Смарт_он'` найдет "Смартфон", "Смарттон" |

### ⚠️ Важно понимать:

- **`%`** может соответствовать **0 символов** (пустая строка)
- **`_`** должен соответствовать **ровно 1 символу**
- Эти символы работают только внутри шаблона LIKE

## Процент (%) - любое количество символов

### `%` в начале шаблона

```sql
-- Товары, заканчивающиеся на "фон"
SELECT * FROM products 
WHERE name LIKE '%фон';
```

**Что найдет:**
- ✅ "Смартфон"
- ✅ "телефон"
- ✅ "фон"
- ❌ "Фонарь" (не заканчивается на "фон")

### `%` в конце шаблона

```sql
-- Товары, начинающиеся с "Смарт"
SELECT * FROM products 
WHERE name LIKE 'Смарт%';
```

**Что найдет:**
- ✅ "Смартфон"
- ✅ "Смартфон 25"
- ✅ "Смарт"
- ❌ "смартфон" (зависит от регистра, см. ниже)

### `%` в начале и конце

```sql
-- Товары, содержащие "Смарт" в любом месте
SELECT * FROM products 
WHERE name LIKE '%Смарт%';
```

**Что найдет:**
- ✅ "Смартфон"
- ✅ "Новый Смартфон"
- ✅ "Смартфон 25"
- ✅ "Смарт"

**Результат:**
| id | name | category |
|----|------|----------|
| 1 | Смартфон 25 | Электроника |
| 15 | Смартфон Pro | Электроника |

### `%` в середине шаблона

```sql
-- Товары, начинающиеся с "Смарт" и заканчивающиеся на "25"
SELECT * FROM products 
WHERE name LIKE 'Смарт%25';
```

**Что найдет:**
- ✅ "Смартфон 25"
- ✅ "Смарт25"
- ✅ "Смарт-устройство-25"
- ❌ "Смартфон 24"

### Несколько `%` в шаблоне

```sql
-- Товары с "Смарт" и "фон" в любом порядке
SELECT * FROM products 
WHERE name LIKE '%Смарт%фон%' OR name LIKE '%фон%Смарт%';
```

**Что найдет:**
- ✅ "Смартфон"
- ✅ "фон Смарт"
- ✅ "Новый Смартфон 25"

## Подчеркивание (_) - один символ

### `_` для одного символа

```sql
-- Товары с названием из 7 символов, начинающимся с "Смарт"
SELECT * FROM products 
WHERE name LIKE 'Смарт__';
```

**Что найдет:**
- ✅ "Смартфон" (7 символов: С-м-а-р-т-ф-о-н, но нужно проверить)
- ❌ "Смарт" (только 5 символов)
- ❌ "Смартфон 25" (больше 7 символов)

**Примечание:** В кириллице нужно учитывать, что один символ = одна позиция.

### Несколько `_` подряд

```sql
-- Товары с форматом "Смарт__ 25" (два любых символа между "Смарт" и " 25")
SELECT * FROM products 
WHERE name LIKE 'Смарт__ 25';
```

**Что найдет:**
- ✅ "Смартфон 25" (если между "Смарт" и " 25" ровно 2 символа)
- ❌ "Смарт 25" (только 1 символ)
- ❌ "Смартфон Pro 25" (больше 2 символов)

### Комбинация `_` и `%`

```sql
-- Товары, где после "Смарт" идет один символ, затем любые символы
SELECT * FROM products 
WHERE name LIKE 'Смарт_%';
```

**Что найдет:**
- ✅ "Смартфон"
- ✅ "Смартфон 25"
- ✅ "СмартX"
- ❌ "Смарт" (нет символа после "Смарт")

## Комбинации символов

### Сложные шаблоны

```sql
-- Товары с артикулом формата ART-XXXXX-YY (5 цифр, дефис, 2 цифры)
SELECT * FROM products 
WHERE article LIKE 'ART-_____-__';
```

**Что найдет:**
- ✅ "ART-12345-01"
- ✅ "ART-99999-99"
- ❌ "ART-1234-01" (только 4 цифры)
- ❌ "ART-123456-01" (6 цифр)

### Поиск по нескольким полям

```sql
-- Товары, где название или описание содержит "качественный"
SELECT * FROM products 
WHERE name LIKE '%качественный%' 
   OR description LIKE '%качественный%';
```

### Поиск с несколькими условиями

```sql
-- Товары категории "Электроника" с "Смарт" в названии
SELECT * FROM products 
WHERE category = 'Электроника' 
  AND name LIKE '%Смарт%';
```

## NOT LIKE - отрицание

**NOT LIKE** возвращает строки, которые **не соответствуют** шаблону.

### Пример 1: Исключить товары с определенным словом

```sql
-- Все товары, кроме тех, что содержат "Смартфон"
SELECT * FROM products 
WHERE name NOT LIKE '%Смартфон%';
```

**Что найдет:**
- ✅ "Ноутбук 12"
- ✅ "Планшет 7"
- ❌ "Смартфон 25"
- ❌ "Новый Смартфон"

### Пример 2: Товары без определенного формата артикула

```sql
-- Товары, артикул которых НЕ начинается с "ART-"
SELECT * FROM products 
WHERE article NOT LIKE 'ART-%';
```

### Пример 3: Комбинация NOT LIKE с другими условиями

```sql
-- Товары категории "Электроника", но не содержащие "Смартфон"
SELECT * FROM products 
WHERE category = 'Электроника' 
  AND name NOT LIKE '%Смартфон%';
```

## Регистр символов

### Важно для SQLite:

В **SQLite** оператор LIKE по умолчанию **не чувствителен к регистру** для ASCII символов, но **чувствителен для Unicode** (кириллица).

### Примеры:

```sql
-- В SQLite это может не сработать для кириллицы
SELECT * FROM products 
WHERE name LIKE 'смартфон%';  -- может не найти "Смартфон"

-- Правильный вариант - учитывать оба регистра
SELECT * FROM products 
WHERE name LIKE '%Смартфон%' OR name LIKE '%смартфон%';

-- Или использовать функции для приведения к одному регистру
SELECT * FROM products 
WHERE LOWER(name) LIKE '%смартфон%';
```

### Функции для работы с регистром:

```sql
-- Приведение к нижнему регистру
SELECT * FROM products 
WHERE LOWER(name) LIKE '%смартфон%';

-- Приведение к верхнему регистру
SELECT * FROM products 
WHERE UPPER(name) LIKE '%СМАРТФОН%';
```

**⚠️ Примечание:** В SQLite функции `LOWER()` и `UPPER()` работают только с ASCII. Для кириллицы лучше использовать оба варианта в OR.

## ESCAPE - экранирование символов

Если нужно найти символы `%` или `_` как обычные символы (не как шаблоны), используется **ESCAPE**.

### Синтаксис:

```sql
WHERE column LIKE 'pattern' ESCAPE 'escape_character';
```

### Пример 1: Поиск символа `%`

```sql
-- Найти товары с описанием, содержащим символ процента
SELECT * FROM products 
WHERE description LIKE '%\%%' ESCAPE '\';
```

**Что происходит:**
- `\%` - экранированный символ процента (ищется как обычный символ)
- `%` в начале и конце - обычные шаблоны LIKE

### Пример 2: Поиск символа `_`

```sql
-- Найти артикулы с подчеркиванием
SELECT * FROM products 
WHERE article LIKE '%\_%' ESCAPE '\';
```

### Пример 3: Поиск обратного слэша

```sql
-- Найти товары с обратным слэшем в названии
SELECT * FROM products 
WHERE name LIKE '%\\%' ESCAPE '\';
```

**Что происходит:**
- `\\` - экранированный обратный слэш (ищется как обычный символ `\`)

### Другой символ экранирования

```sql
-- Использование другого символа для экранирования
SELECT * FROM products 
WHERE description LIKE '%#_%' ESCAPE '#';
```

## LIKE с другими операторами

### LIKE с IN

```sql
-- Товары, название которых соответствует одному из шаблонов
SELECT * FROM products 
WHERE name LIKE '%Смартфон%' 
   OR name LIKE '%Ноутбук%'
   OR name LIKE '%Планшет%';

-- Эквивалентно с использованием подзапроса (если нужно)
SELECT * FROM products 
WHERE name IN (
    SELECT name FROM products 
    WHERE name LIKE '%Смартфон%' 
       OR name LIKE '%Ноутбук%'
);
```

### LIKE с BETWEEN

```sql
-- Товары с определенным форматом артикула и ценой в диапазоне
SELECT * FROM products 
WHERE article LIKE 'ART-%' 
  AND price BETWEEN 10000 AND 20000;
```

### LIKE с агрегатными функциями

```sql
-- Количество товаров, содержащих "Смартфон" в названии
SELECT COUNT(*) AS count
FROM products 
WHERE name LIKE '%Смартфон%';

-- Группировка по категориям с LIKE
SELECT 
    category,
    COUNT(*) AS total,
    COUNT(CASE WHEN name LIKE '%Смартфон%' THEN 1 END) AS smartphones_count
FROM products
GROUP BY category;
```

### LIKE в подзапросах

```sql
-- Товары категории "Электроника", название которых похоже на другие товары
SELECT * FROM products p1
WHERE p1.category = 'Электроника'
  AND EXISTS (
      SELECT 1 FROM products p2
      WHERE p2.id != p1.id
        AND p2.name LIKE '%' || SUBSTR(p1.name, 1, 5) || '%'
  );
```

## Производительность LIKE

### ⚠️ Важно для производительности:

1. **LIKE с `%` в начале медленный**
   ```sql
   -- МЕДЛЕННО (не может использовать индекс)
   WHERE name LIKE '%Смартфон';
   
   -- БЫСТРЕЕ (может использовать индекс, если он есть)
   WHERE name LIKE 'Смартфон%';
   ```

2. **Индексы и LIKE**
   - Индексы работают только если шаблон начинается с конкретного текста
   - `LIKE 'Смартфон%'` - может использовать индекс
   - `LIKE '%Смартфон'` - не может использовать индекс
   - `LIKE '%Смартфон%'` - не может использовать индекс

3. **Создание индекса для ускорения**
   ```sql
   -- Создать индекс для ускорения поиска
   CREATE INDEX idx_products_name ON products(name);
   
   -- Теперь этот запрос будет быстрее
   SELECT * FROM products WHERE name LIKE 'Смартфон%';
   ```

### Рекомендации:

- ✅ Используйте `LIKE 'text%'` вместо `LIKE '%text%'` когда возможно
- ✅ Создавайте индексы на столбцы, по которым часто ищете
- ✅ Для полнотекстового поиска рассмотрите FTS (Full-Text Search)

## Альтернативы LIKE

### 1. GLOB (только SQLite)

**GLOB** похож на LIKE, но использует синтаксис оболочки Unix:

```sql
-- GLOB чувствителен к регистру
SELECT * FROM products 
WHERE name GLOB '*Смартфон*';

-- GLOB использует * вместо %, ? вместо _
SELECT * FROM products 
WHERE article GLOB 'ART-?????-??';
```

### 2. REGEXP (в некоторых СУБД)

```sql
-- Регулярные выражения (если поддерживается)
SELECT * FROM products 
WHERE name REGEXP '^Смарт.*фон$';
```

**⚠️ Примечание:** SQLite не поддерживает REGEXP по умолчанию, но можно добавить через расширения.

### 3. INSTR (поиск подстроки)

```sql
-- INSTR возвращает позицию подстроки (0 если не найдено)
SELECT * FROM products 
WHERE INSTR(name, 'Смартфон') > 0;
```

**Преимущества:**
- Работает быстрее, чем `LIKE '%text%'`
- Не зависит от специальных символов

### 4. SUBSTR (извлечение подстроки)

```sql
-- Проверка начала строки
SELECT * FROM products 
WHERE SUBSTR(name, 1, 6) = 'Смарт';
```

## Практические примеры

### Пример 1: Поиск товаров по части названия

```sql
-- Найти все товары, содержащие "фон" в названии
SELECT id, name, category, price
FROM products
WHERE name LIKE '%фон%'
ORDER BY name;
```

### Пример 2: Поиск по артикулу с определенным форматом

```sql
-- Найти товары с артикулом ART-XXXXX-1 (заканчивающиеся на -1)
SELECT * FROM products
WHERE article LIKE 'ART-_____-1';
```

### Пример 3: Поиск товаров по категории и названию

```sql
-- Товары категории "Электроника", начинающиеся с "Смарт"
SELECT name, price
FROM products
WHERE category = 'Электроника'
  AND name LIKE 'Смарт%'
ORDER BY price DESC;
```

### Пример 4: Поиск в описании

```sql
-- Товары с описанием, содержащим "качественный" или "надежный"
SELECT name, description
FROM products
WHERE description LIKE '%качественный%'
   OR description LIKE '%надежный%';
```

### Пример 5: Исключение определенных товаров

```sql
-- Все товары, кроме смартфонов и ноутбуков
SELECT name, category, price
FROM products
WHERE name NOT LIKE '%Смартфон%'
  AND name NOT LIKE '%Ноутбук%'
ORDER BY category, name;
```

### Пример 6: Поиск с несколькими условиями

```sql
-- Товары категории "Электроника" или "Компьютеры",
-- название содержит "Смарт" или "Планшет",
-- цена больше 10000
SELECT name, category, price
FROM products
WHERE (category = 'Электроника' OR category = 'Компьютеры')
  AND (name LIKE '%Смарт%' OR name LIKE '%Планшет%')
  AND price > 10000
ORDER BY price;
```

### Пример 7: Поиск по формату артикула

```sql
-- Товары с артикулом формата ART-XXXXX (5 цифр после ART-)
SELECT article, name, price
FROM products
WHERE article LIKE 'ART-_____'
ORDER BY article;
```

### Пример 8: Поиск товаров с определенным количеством символов

```sql
-- Товары с названием ровно из 8 символов
SELECT name, LENGTH(name) AS name_length
FROM products
WHERE name LIKE '________'
ORDER BY name;
```

**Примечание:** `LENGTH()` возвращает количество символов в строке.

### Пример 9: Поиск с использованием подзапроса

```sql
-- Товары, название которых похоже на название других товаров
SELECT DISTINCT p1.name, p1.category
FROM products p1
WHERE EXISTS (
    SELECT 1 FROM products p2
    WHERE p2.id != p1.id
      AND p2.name LIKE '%' || SUBSTR(p1.name, 1, 4) || '%'
)
ORDER BY p1.name;
```

### Пример 10: Статистика по шаблонам

```sql
-- Подсчет товаров по шаблонам названий
SELECT 
    CASE
        WHEN name LIKE '%Смартфон%' THEN 'Смартфоны'
        WHEN name LIKE '%Ноутбук%' THEN 'Ноутбуки'
        WHEN name LIKE '%Планшет%' THEN 'Планшеты'
        WHEN name LIKE '%Наушники%' THEN 'Наушники'
        ELSE 'Другое'
    END AS product_type,
    COUNT(*) AS count,
    AVG(price) AS avg_price
FROM products
GROUP BY product_type
ORDER BY count DESC;
```

### Пример 11: Поиск с экранированием

```sql
-- Если в базе есть товары с символом % в названии
-- (например, "Скидка 50%")
SELECT * FROM products
WHERE name LIKE '%\%%' ESCAPE '\';
```

### Пример 12: Комплексный поиск

```sql
-- Поиск товаров по нескольким критериям с LIKE
SELECT 
    p.name,
    p.category,
    p.price,
    p.description
FROM products p
WHERE (
    -- Поиск в названии
    p.name LIKE '%Смарт%' 
    OR p.name LIKE '%фон%'
)
AND (
    -- Поиск в описании
    p.description LIKE '%качественный%'
    OR p.description LIKE '%надежный%'
    OR p.description IS NULL
)
AND p.price BETWEEN 10000 AND 30000
ORDER BY p.price DESC
LIMIT 20;
```

## Частые ошибки и как их избежать

### ❌ Ошибка 1: Забыли кавычки

```sql
-- НЕПРАВИЛЬНО
WHERE name LIKE %Смартфон%;

-- ПРАВИЛЬНО
WHERE name LIKE '%Смартфон%';
```

### ❌ Ошибка 2: Путаница между % и _

```sql
-- НЕПРАВИЛЬНО (ищет ровно один символ)
WHERE article LIKE 'ART-_-1';

-- ПРАВИЛЬНО (ищет любое количество символов)
WHERE article LIKE 'ART-%-1';
```

### ❌ Ошибка 3: Регистр для кириллицы

```sql
-- Может не сработать в SQLite
WHERE name LIKE 'смартфон%';

-- ПРАВИЛЬНО (учитывать оба варианта)
WHERE name LIKE '%Смартфон%' OR name LIKE '%смартфон%';
-- или
WHERE LOWER(name) LIKE '%смартфон%';
```

### ❌ Ошибка 4: Поиск специальных символов без ESCAPE

```sql
-- НЕПРАВИЛЬНО (будет искать шаблон, а не символ %)
WHERE description LIKE '%50%%';

-- ПРАВИЛЬНО
WHERE description LIKE '%50\%%' ESCAPE '\';
```

## Заключение

Оператор LIKE - мощный инструмент для поиска по шаблонам:

✅ **Гибкость:** поиск по частичному совпадению  
✅ **Простота:** интуитивно понятный синтаксис  
✅ **Универсальность:** работает с любыми текстовыми данными  

**Помните:**
- Используйте `%` для любого количества символов
- Используйте `_` для одного символа
- Будьте осторожны с регистром (особенно для кириллицы)
- Используйте ESCAPE для поиска специальных символов
- Создавайте индексы для ускорения поиска
- Избегайте `LIKE '%text%'` когда возможно - используйте `LIKE 'text%'`

Правильное использование LIKE значительно упрощает поиск и фильтрацию данных в базе.
