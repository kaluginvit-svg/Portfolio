# Time API — тестовый бэкенд на FastAPI

Простое API, возвращающее текущее время сервера.

## Запуск

Из **корня проекта** (там, где лежит `main.py`):

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск сервера (слушает все интерфейсы, порт 8080)
uvicorn main:app --reload --host 0.0.0.0 --port 8080
```

После запуска откройте в браузере:
- **http://127.0.0.1:8080** или **http://localhost:8080**
- Документация: **http://127.0.0.1:8080/docs**

Если не открывается:
1. Запускайте команду из папки, где лежит `main.py`.
2. Убедитесь, что порт 8080 свободен (в другом терминале: `netstat -ano | findstr 8080` — пусто).
3. Попробуйте в браузере и **http://localhost:8080**, и **http://127.0.0.1:8080**.

## Эндпоинты

| Метод | Путь        | Описание                          |
|-------|-------------|-----------------------------------|
| GET   | `/`         | Приветствие и ссылка на документацию |
| GET   | `/time`     | Текущее время по UTC              |
| GET   | `/datetime` | Полная дата и время по UTC        |
| GET   | `/date`     | Текущая дата по UTC               |
| GET   | `/convert`  | Конвертация времени по часовым поясам |
| GET   | `/health`   | Проверка работоспособности        |

Документация (Swagger UI): **http://127.0.0.1:8080/docs**

## Примеры ответов

**GET /time**
```json
{
  "time": "2025-03-12T14:30:00.123456+00:00",
  "timestamp": 1710246600.123456
}
```

**GET /convert?from_tz=UTC&to_tz=Europe/Moscow** — конвертация текущего времени из UTC в Москву:
```json
{
  "from_tz": "UTC",
  "to_tz": "Europe/Moscow",
  "original": "2025-03-12T14:30:00+00:00",
  "converted": "2025-03-12T17:30:00+03:00",
  "timestamp": 1710250200.0
}
```
Можно передать `dt` в ISO 8601 для конвертации конкретного момента.

## GitHub Actions: сборка и деплой

Workflow `.github/workflows/deploy.yml`:

1. **build-and-push** — собирает Docker-образ и пушит в **GitHub Container Registry** (`ghcr.io/<owner>/<repo>`).
2. **deploy** — по SSH подключается к серверу, логинится в GHCR, скачивает образ из реестра и запускает контейнер на порту 8080.

Запуск: при пуше в `main` или вручную (workflow_dispatch). Подробнее — в [DEPLOY.md](DEPLOY.md).

### Секреты (Settings → Secrets and variables → Actions)

| Секрет | Описание |
|--------|----------|
| `GHCR_TOKEN` | GitHub PAT (classic) с правом **read:packages** (для pull на сервере) |
| `GHCR_USERNAME` | Логин GitHub владельца токена |
| `SSH_HOST` | IP или hostname сервера |
| `SSH_USER` | Пользователь для SSH |
| `SSH_PRIVATE_KEY` | Приватный SSH-ключ (полностью, с `-----BEGIN ... -----`) |
| `SSH_PORT` | Порт SSH (необязательно, по умолчанию 22) |
