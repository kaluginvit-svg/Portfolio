# FastAPI Service Generator via LangChain

## Project Purpose

A sophisticated pipeline that leverages LangChain and LLM to automatically generate production-ready FastAPI services from natural language descriptions. The system performs multi-stage analysis, planning, code generation, and validation to produce functional REST APIs without manual intervention.

## 4-Stage Pipeline

### Stage 1: Task Analysis
- Analyzes the input task description
- Extracts key information about the service:
  - Service name and main goal
  - Data entities and relationships
  - Required API endpoints
  - Request/response schemas
  - Database and authentication requirements
  - Business rules and validation rules
  - Edge cases and error scenarios
- Returns structured JSON output for downstream stages

### Stage 2: Tool Selection
- Determines optimal technology stack based on analysis
- Selects frameworks (FastAPI, Pydantic)
- Chooses ORM if database is needed (SQLAlchemy)
- Selects appropriate database (SQLite, PostgreSQL)
- Identifies additional dependencies
- Provides reasoning for tool choices
- Returns JSON configuration for code generation

### Stage 3: Code Generation
- Generates complete, production-ready FastAPI service code
- Creates Pydantic models for request/response validation
- Implements all required endpoints
- Adds database models and operations if needed
- Includes error handling and type hints
- Ensures code is executable without modifications
- Includes main execution block with uvicorn server

### Stage 4: Code Review & Validation
- Validates Python syntax using AST parsing
- Checks for missing imports and app initialization
- Verifies all endpoints are properly defined
- Confirms request/response models exist
- Validates main execution block presence
- Corrects any identified issues
- Final syntax validation before saving

## Technology Stack

- **Language**: Python 3.11+
- **Framework**: FastAPI
- **Validation**: Pydantic v2
- **Database ORM**: SQLAlchemy (when needed)
- **Database**: SQLite (default) or PostgreSQL
- **Server**: Uvicorn
- **Chain Orchestration**: LangChain
- **LLM Integration**: OpenAI-compatible API
- **Environment Management**: python-dotenv
- **Code Validation**: ast module

## Installation

1. Clone or create the project directory:
```bash
mkdir fastapi-generator
cd fastapi-generator
```

2. Create a Python virtual environment:
```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

3. Install dependencies:
```bash
pip install -r requirements.txt
```

## Configuration

### Step 1: Create .env file
Copy the example configuration:
```bash
cp .env.example .env
```

### Step 2: Update .env with your settings
Edit `.env` and set your values:

```env
# API Key for your LLM provider
PROXY_API=your-actual-api-key-here

# Base URL for OpenAI-compatible LLM endpoint
URL=https://api.openai.com/v1

# Model name to use
MODEL_NAME=gpt-4

# Temperature for response generation (0.0-2.0)
TEMPERATURE=0.2

# Logging level (DEBUG, INFO, WARNING, ERROR, CRITICAL)
LOG_LEVEL=INFO
```

### Example .env Configuration

```
PROXY_API=sk-proj-abc123xyz789def456ghi789jkl012mno345pqr
URL=https://api.openai.com/v1
MODEL_NAME=gpt-4-turbo
TEMPERATURE=0.2
LOG_LEVEL=INFO
```

For alternative LLM providers (Ollama, Local LPLMs, etc.):
```
PROXY_API=local-key
URL=http://localhost:8000/v1
MODEL_NAME=local-model-name
TEMPERATURE=0.3
LOG_LEVEL=DEBUG
```

## Usage

### Basic Execution

Run the pipeline with a task description:

```bash
python script.py "Create a task management API with users and tasks"
```

### Example Command

```bash
python script.py "Build a blog API with posts, comments, and user authentication. Include endpoints for creating, reading, updating, and deleting posts. Posts should have title, content, author, and creation date. Comments need post_id and author fields."
```

### What Happens During Execution

1. **Task Analysis** - Extracts requirements from your description
2. **Tool Selection** - Determines the best technology choices
3. **Code Generation** - Creates the full FastAPI application
4. **Code Review** - Validates and fixes any issues
5. **Output Saved** - Complete service saved to `generated_api/main.py`

## Generated Output

### File: `generated_api/main.py`

This is the complete, production-ready FastAPI service generated by the pipeline.

**What's Included:**
- FastAPI application initialization
- Pydantic model definitions for request/response validation
- Complete endpoint implementations matching your requirements
- Database models and initialization (if database was needed)
- Error handling and logging
- Type hints throughout
- Proper imports
- Main execution block for running with Uvicorn

**To Run the Generated Service:**

```bash
python generated_api/main.py
```

The service will start on `http://localhost:8000`

**API Documentation:**
- Swagger UI: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`

## Logging

The pipeline provides detailed logging at each stage.

### Log Levels

- **DEBUG**: Detailed information for debugging
- **INFO**: General informational messages (default)
- **WARNING**: Warning messages
- **ERROR**: Error messages
- **CRITICAL**: Critical issues

### Log Output Format

```
2024-01-15 10:30:45,123 - __main__ - INFO - Stage 1: Analyzing task...
2024-01-15 10:30:47,456 - __main__ - INFO - Task analysis completed: blog_service
2024-01-15 10:30:48,789 - __main__ - INFO - Stage 2: Selecting tools...
```

### View Logs

Change `LOG_LEVEL` in .env to see more detailed output:

```env
LOG_LEVEL=DEBUG  # For detailed execution logs
LOG_LEVEL=INFO   # For normal operation
```
