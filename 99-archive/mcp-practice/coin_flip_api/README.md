# 🪙 Coin Flip API

API для подбрасывания монетки на Python (Flask) с веб-интерфейсом.

> ⚠️ **Важно:** Этот проект предназначен **только для запуска в Docker контейнере через Docker Compose**. Локальный запуск не поддерживается.

## 🚀 Быстрый старт

### Требования

- Docker Desktop (Windows/Mac) или Docker Engine (Linux)
- Docker Compose (входит в Docker Desktop)

### Запуск в терминале Cursor

1. Откройте терминал: `Ctrl + ~`
2. Убедитесь, что вы в папке проекта
3. Выполните:

```bash
docker-compose up -d --build
```

4. Откройте в браузере: **http://localhost:5000**

> 💡 **Примечание:** Команды `docker-compose` работают одинаково в PowerShell, CMD и Linux/Mac терминалах.

## 🌐 Использование

После запуска контейнера откройте в браузере: **http://localhost:5000**

### Визуальный интерфейс

На главной странице доступен веб-интерфейс с:
- 🪙 Анимированной монеткой
- 🔘 Кнопками для подбрасывания (1, 5, 10, 20 раз)
- 📊 Статистикой в реальном времени
- 📜 Историей последних результатов (до 50)
- 💾 Автосохранением статистики в браузере

**Как использовать:**
1. Нажмите любую кнопку подбрасывания
2. Монетка анимируется и покажет результат
3. Статистика обновится автоматически
4. Результат добавится в историю

### API Эндпоинты

- `GET /` - Веб-интерфейс
- `GET /flip` - Подбросить монетку один раз (JSON)
- `GET /flip/<count>` - Подбросить монетку несколько раз (от 1 до 100)

### Примеры запросов

```bash
# Подбросить монетку один раз
curl http://localhost:5000/flip

# Подбросить монетку 10 раз
curl http://localhost:5000/flip/10
```

### Пример ответа

**Одно подбрасывание:**
```json
{
  "result": "орел",
  "flip": 1
}
```

**Несколько подбрасываний:**
```json
{
  "results": ["орел", "решка", "орел", "решка", "орел"],
  "statistics": {
    "total": 5,
    "орел": 3,
    "решка": 2
  }
}
```

## 🛑 Управление контейнером

### Основные команды

| Действие | Команда | Когда использовать |
|----------|---------|-------------------|
| **Запуск** | `docker-compose up -d --build` | Первый запуск или после изменений кода |
| **Запуск с логами** | `docker-compose up --build` | Когда нужно видеть что происходит |
| **Остановка** | `docker-compose down` | Когда закончили работу |
| **Перезапуск** | `docker-compose restart` | Быстрый перезапуск без пересборки |
| **Просмотр логов** | `docker-compose logs -f` | Отладка проблем |
| **Статус** | `docker-compose ps` | Проверка что контейнер работает |
| **Пересборка** | `docker-compose build --no-cache` | Полная пересборка образа |

### Продвинутые команды

**Просмотр использования ресурсов:**
```bash
docker stats coin-flip-api
```

**Информация о контейнере:**
```bash
docker inspect coin-flip-api
```

**Вход в контейнер (для отладки):**
```bash
docker exec -it coin-flip-api /bin/bash
```

**Проверка здоровья:**
```bash
docker-compose ps
# В колонке STATUS будет видно (healthy) или (unhealthy)
```

## 🆘 Решение проблем

### Порт 5000 занят

**Вариант 1:** Остановите другие контейнеры
```bash
docker-compose down
```

**Вариант 2:** Измените порт в `docker-compose.yml`:
```yaml
ports:
  - "5001:5000"  # Внешний порт 5001, внутренний 5000
```

### Контейнер не запускается

**Проверьте логи:**
```bash
docker-compose logs
```

**Пересоберите образ:**
```bash
docker-compose build --no-cache
docker-compose up -d
```

### Контейнер постоянно перезапускается

**Проверьте логи:**
```bash
docker-compose logs --tail=50
```

**Проверьте healthcheck:**
```bash
docker inspect coin-flip-api | grep -A 10 Health
```

### Полная очистка

**Удалить контейнер и образ:**
```bash
docker-compose down
docker rmi coin-flip-api:latest
```

**Очистить все неиспользуемые ресурсы Docker:**
```bash
docker system prune -a
```
