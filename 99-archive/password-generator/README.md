# CLI менеджер паролей на Python

Приложение хранит учётные данные в SQLite и шифрует пароли с помощью `cryptography.Fernet`. Мастер-пароль хэшируется (SHA-256) и требуется при каждом запуске (кроме генерации нового пароля).

## Требования
- Python 3.9+
- Пакет `cryptography`

## Установка и запуск в виртуальном окружении
Ниже пример для Windows PowerShell; для bash/zsh замените активацию на `source .venv/bin/activate`.

```pwsh
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Использование
Запускать из корня проекта:
```pwsh
python password_manager.py
```

Далее приложение работает в диалоговом режиме (меню на русском; английские варианты команд тоже принимаются). Доступные действия:
- `добавить` — сохранить пароль: ввод названия и логина, затем предложение сгенерировать пароль (с выбором длины и типов символов) или ввести свой.
- `получить` — показать логин и пароль по названию.
- `список` — показать все названия и логины.
- `удалить` — удалить пароль по названию (с подтверждением).
- `новый` — сгенерировать новый пароль (можно указать длину, по умолчанию 16; минимум 8).

## Графическое приложение (Windows)
Для запуска GUI-версии (Tkinter):
```pwsh
python password_manager_gui.py
```
Функционал: ввод/проверка мастер-пароля, список паролей, добавление (с генератором и выбором типов символов), просмотр/копирование в буфер, удаление.

Примеры:
```pwsh
python password_manager.py add Google user@example.com
python password_manager.py get Google
python password_manager.py list
python password_manager.py delete Google
python password_manager.py new -l 20
```

## Как это работает
- При первом запуске запросит мастер-пароль и сохранит его хэш в таблице `settings`.
- Ключ шифрования хранится в файле `.key` (создаётся при первом запуске); база — в `vault.db`.
- Таблицы `credentials` и `settings` создаются автоматически при старте.

## Советы по безопасности
- Храните `.key` и `vault.db` вместе и не коммитьте их в git.
- Используйте длинный мастер-пароль и делайте резервные копии файла ключа и базы.
