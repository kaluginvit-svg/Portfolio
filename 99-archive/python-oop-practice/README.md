# Система управления проектами и задачами

Полнофункциональная система управления проектами и задачами, реализованная на Python с использованием принципов ООП.

## 📁 Структура проекта

```
ООП/
├── models/                 # Модели данных
│   ├── __init__.py        # Экспорт всех моделей
│   ├── enums.py           # Перечисления (TaskStatus, ProjectRole)
│   ├── user.py            # Модель пользователя
│   ├── project.py         # Модель проекта
│   ├── task.py            # Модель задачи
│   └── membership.py      # Модель членства в проекте
│
├── services/              # Сервисный слой (бизнес-логика)
│   ├── __init__.py        # Экспорт всех сервисов
│   ├── project_service.py # Сервис управления проектами
│   └── task_service.py    # Сервис управления задачами
│
├── main.py                # Точка входа и демонстрация
├── arhitecture.md         # Описание архитектуры системы
└── README.md              # Этот файл
```

## 🏗️ Архитектура

Система построена на основе **архитектурного паттерна с сервисным слоем**:

- **Модели данных** - простые классы для хранения данных (User, Project, Task, Membership)
- **Сервисы** - содержат всю бизнес-логику и проверки прав доступа

### Модели данных

#### `User` (Пользователь)
- **Атрибуты**: id, username, email, created_at
- **Методы**: update_profile()

#### `Project` (Проект)
- **Атрибуты**: id, name, description, owner_id, created_at, is_archived
- **Методы**: archive(), change_name()

#### `Task` (Задача)
- **Атрибуты**: id, title, description, status, due_date, project_id, assignee_id, created_at
- **Методы**: mark_as_done(), reopen(), set_due_date()

#### `Membership` (Членство)
- **Атрибуты**: user_id, project_id, role
- **Описание**: Связывает пользователя с проектом и определяет его роль

### Сервисы

#### `ProjectService`
Управляет проектами и правами доступа:
- `create_project()` - создание проекта
- `add_user_to_project()` - добавление участника (с проверкой прав)
- `get_project_tasks()` - получение задач (с проверкой доступа)
- `get_user_role()` - получение роли пользователя

#### `TaskService`
Управляет жизненным циклом задач:
- `create_task()` - создание задачи (с проверкой членства)
- `assign_task()` - назначение ответственного (с проверкой прав)
- `change_task_status()` - изменение статуса (с проверкой доступа)

## 🚀 Запуск

```bash
python main.py
```

Программа запустит демонстрацию всех возможностей системы:
1. Создание пользователей
2. Создание проектов
3. Управление участниками
4. Создание и управление задачами
5. Работу системы прав доступа

## 🔐 Система прав доступа

- **Admin** - может добавлять участников в проект
- **Member** - может создавать задачи и назначать ответственных
- **Guest** - может просматривать задачи

### Проверки безопасности:
- ✅ Только администраторы могут добавлять участников
- ✅ Только участники проекта могут создавать задачи
- ✅ Нельзя назначить ответственным пользователя вне проекта
- ✅ Пользователи вне проекта не могут видеть его задачи

## 💡 Примеры использования

### Создание пользователя
```python
from models import User

alice = User(username="Alice", email="alice@example.com")
```

### Создание проекта
```python
from services import ProjectService

project_service = ProjectService()
project = project_service.create_project(
    user=alice,
    name="Мой проект",
    description="Описание проекта"
)
```

### Добавление участника
```python
from models import ProjectRole

project_service.add_user_to_project(
    acting_user=alice,      # Должен быть администратором
    project=project,
    target_user=bob,
    role=ProjectRole.MEMBER
)
```

### Создание задачи
```python
from services import TaskService

task_service = TaskService()
task = task_service.create_task(
    user=alice,              # Должен быть участником проекта
    project=project,
    project_service=project_service,
    title="Важная задача",
    description="Детальное описание"
)
```

## 📊 Особенности реализации

1. **UUID идентификаторы** - для уникальности объектов
2. **Enum для статусов и ролей** - типобезопасность
3. **Проверка прав доступа** - на уровне сервисов
4. **Разделение ответственности** - модели для данных, сервисы для логики
5. **Поддержка Windows** - корректная работа с кириллицей
